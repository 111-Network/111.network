# Plan: bulk-copy `_template/` into root

## Context

Root is a fresh Next.js 16.2.6 install with only a minimal `src/app/{layout,page,globals.css}` and 5 stock SVGs in `public/`. `_template/` is a standalone Next.js 16.1.6 ASCII-hub design with a full shadcn/ui kit (57 components), an `ascii-hub` section system, and accompanying hooks/lib/styles. We want to absorb the template into the root as the new app surface with **command-copy**, not file-by-file rewriting.

**Goals:**
- Use `cp -r` to move template trees, with **zero code edits inside the copied files**.
- Keep pnpm.
- Keep the root's `src/` layout (so template trees land under `src/`).
- Carry all files including unused shadcn blocks and duplicates (per user instruction).
- After copy, the project boots and renders the template's home page.

**Why no code edits work:** template imports use `@/components/...`, `@/lib/utils`, `@/hooks/...`. Root tsconfig maps `@/*` → `./src/*`. Copying template's `components/`, `lib/`, `hooks/` into `src/` keeps every import path valid as-is.

**Outside scope:** `_outdated-web/` and the legacy site migration are NOT covered here. Tracked separately in `docs/migration-plan.md`.

## Decisions (locked)

1. **Layout:** keep `src/`. Template `app/`, `components/`, `hooks/`, `lib/`, `styles/` → land under `src/`.
2. **`next.config`:** overwrite root `next.config.ts` with template `next.config.mjs`. Delete `.ts`.
3. **Duplicates:** copy verbatim — keeps `styles/globals.css` (shadcn default neutral palette) and the `hooks/use-mobile.ts` + `components/ui/use-mobile.tsx` pair.

## File map (template → root)

| Template path | Root target | Action | Notes |
|---|---|---|---|
| `_template/app/globals.css` | `src/app/globals.css` | overwrite | template's dark OKLCH + `@theme inline` |
| `_template/app/layout.tsx` | `src/app/layout.tsx` | overwrite | |
| `_template/app/page.tsx` | `src/app/page.tsx` | overwrite | |
| `_template/components/` | `src/components/` | new dir | 1 root file + `ascii-hub/`(9+8) + `ui/`(57) |
| `_template/hooks/` | `src/hooks/` | new dir | `use-mobile.ts`, `use-toast.ts` |
| `_template/lib/` | `src/lib/` | new dir | `sections-data.ts`, `utils.ts` |
| `_template/styles/` | `src/styles/` | new dir | duplicate `globals.css` (unused) |
| `_template/components.json` | `components.json` | new file | **edit** one line: `tailwind.css: "src/app/globals.css"` |
| `_template/next.config.mjs` | `next.config.mjs` | new file | delete root `next.config.ts` first |
| `_template/postcss.config.mjs` | `postcss.config.mjs` | overwrite | functionally identical, harmless |

## Files to NOT copy

| Template path | Why |
|---|---|
| `package.json` | merge deps with `pnpm add`; preserve root's pnpm setup |
| `pnpm-lock.yaml` | regenerated by `pnpm install` |
| `tsconfig.json` | root's `@/*` → `./src/*` mapping must be kept |
| `next-env.d.ts` | Next regenerates |
| `.gitignore` | root has its own |
| `tsconfig.tsbuildinfo` | build artifact |

## Files to delete at root after copy

- `src/app/favicon.ico` — replaced by metadata in `layout.tsx`
- `public/{file,globe,next,vercel,window}.svg` — stock Next placeholders, remove to keep `public/` clean for new media
- `next.config.ts` — replaced by `.mjs`

## Commands (executable in this order)

```bash
# 1. Remove stuff being replaced
rm src/app/favicon.ico
rm public/file.svg public/globe.svg public/next.svg public/vercel.svg public/window.svg
rm next.config.ts

# 2. Bulk copy template trees (media files intentionally skipped — see below)
cp _template/app/globals.css   src/app/globals.css
cp _template/app/layout.tsx    src/app/layout.tsx
cp _template/app/page.tsx      src/app/page.tsx
cp -r _template/components     src/components
cp -r _template/hooks          src/hooks
cp -r _template/lib            src/lib
cp -r _template/styles         src/styles
cp _template/components.json   components.json
cp _template/next.config.mjs   next.config.mjs
cp _template/postcss.config.mjs postcss.config.mjs

# 3. Patch components.json so shadcn knows where the CSS lives
```

## Critical files (post-copy)

- `src/app/page.tsx` — entry; renders ASCII hub
- `src/app/layout.tsx` — uses `next-themes` ThemeProvider + Geist font
- `src/app/globals.css` — Tailwind v4 + `@theme inline` + `tw-animate-css`
- `components.json` — shadcn config (the **one** post-copy edit)
- `next.config.mjs` — replaces `.ts`
- `package.json` — gains ~50 deps; pnpm is preserved

## Verification

1. `pnpm install` — lockfile resolves cleanly, no peer warnings beyond known noise.
2. `pnpm dev` — starts on port 3000.
3. Open `http://localhost:3000/`:
   - Page renders ASCII canvas + hero + navigation + section blocks + footer.
   - No console errors about missing modules or unresolved `@/*` imports.
   - Tailwind styles apply (dark background, monospace font).
   - Image/icon references may 404 until you add your own media files to `public/`.
4. `pnpm build` — type-check passes, no missing dep errors.
5. Spot-check imports resolve:
   - `grep -r "@/components/ui/button" src/` → resolves to `src/components/ui/button.tsx` ✓
   - `grep -r "@/lib/utils" src/` → resolves to `src/lib/utils.ts` ✓
   - `grep -r "@/hooks/use-mobile" src/` → resolves to `src/hooks/use-mobile.ts` ✓

## Media files — intentionally skipped

Template `public/` (9 images: `og-image.jpg`, `profile-icon.png`, `apple-icon.png`, `icon.svg`, `images/site-logo-*.png`, etc.) is **not copied**. You will create new media assets and place them directly in `public/` yourself. Any references in the copied code to these files will need the assets to exist before they render correctly.

## Risks & follow-ups (out of scope for this plan)

- Template Next 16.1.6 vs root 16.2.6: features used are compatible; if a code-mod is needed it's tracked separately.
- `geist` font import in `layout.tsx` may need a small change if the API differs across versions — verify on first `pnpm dev`.
- `styles/globals.css` is now an orphan (shadcn points to `src/app/globals.css`); cleanup in a later pass.
- After this lands, `_template/` itself can be deleted — defer until verification passes.
